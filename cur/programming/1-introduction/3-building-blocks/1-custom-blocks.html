Bis jetzt habt ihr immer Blöcke verwendet, die bereits existiert haben. Jetzt werdet ihr lernen, wie ihr eure eigenen Blöcke erstellen könnt und diese benutzen könnt.

Ihr werdet mit dem bereits bekannten Skript (siehe rechts) euren eigenen zeichne Quadrat Block definieren. Dann werdet ihr den zeichne Dreieck Block, dann den zeichne Haus Block und schließlich den zeichne Reihe von Häusern Block erstellen.

Durch das Erstellen eines Blockes wird dem Skript ein Name zugewiesen, der innerhalb eines anderen Skriptes für dieses Skript steht. Das ist eine Form der Abstraktion. Hierbei wird hinter einer Bezeichnung der entsprechende Code versteckt, um dem Skript eine klare Struktur zu geben und es dadurch einfacher auf Fehler zu untersuchen (zu debuggen). Später werdet ihr lernen wie ihr eurem Block eine Eingabe Variable gebt, wie bei dem Block, der eine Eingabe bekommt, damit der Sprite weiß wie weit er sich bewegen soll.

Jeder Block braucht einen Platz zum "leben": eine Palette (ausgedrückt durch eine Farbe), einen Titel, eine Kategorie (ausgedrückt durch die Form des Blockes) und einem Skript, das sein Verhalten definiert.
  • Ihr könnt euren Block in jede Palette einordnen, die für euch Sinn macht. Falls ihr keine Palette/Farbe ausgewählt habt, erscheint der Block grau und gehört zu der Palette "Andere" und wird zu den Variablen gezählt.
    Palette: Wähle eine Palette für euren neuen Block aus. ("Bewegung" oder "Stift" wären passende Paletten für den zeichne Quadrat Block.)
    • Ihr könnt weitere Informationen in der Snap Anleitung (S.17-18) bekommen.
  • Titel: Gebt eurem Block einen Namen. Für diesen Block tippt wie hier gezeigt "zeichne Quadrat" ein.
  • Kategorie: Lasst "Befehl" ausgewählt, da dieser Block eine Aktion (Zeichnen eines Quadrats) ausführt, anstatt einen Wert zurückzugeben.
  • Für alle Sprites. In diesem Kurs solltet ihr fast immer "für alle Sprites" ausgewählt lassen, so dass ihr diese Wahl vorerst ignorieren könnt, und auf "OK" klicken könnt.
  • Behavior: A dialog titled "Block Editor" will appear. Build the instructions for your new block here just as you do in the scripting area. Drag blocks from the palette to define the script that will run when your custom block is used. When you are finished, click OK or Apply.
    • The editor starts you with a hat block that contains a picture of your block. Remember that hat blocks say when the script below them should run: when flag clicked, when space key pressed, etc. This hat block says, "when the draw square block is used." (Don't worry about the + signs for the moment; they just let you add more words or inputs.)

Practice building blocks.

    In 1.3.1.4 we show a video of someone drawing the turning at the corner of an equilateral triangle. But we gave away the story in 1.2.4. --AC
  1. Define a draw triangle block that draws an equilateral triangle with the same side length as the square you just drew (50). (Don't click this sentence unless you've been working on this for ten solid minutes and need a hint.)
  2. Create a draw house block that uses your draw square and draw triangle blocks together to draw a "house" shape: a square with a triangular shaped roof sitting on top. (You might have to do some debugging to get the two shapes to connect properly.)
  3. Build a draw row of houses block that draws a row of attached houses, each new one to the right of the previous, until the sprite moves too close to the right edge of the screen. For that, you will need repeat until.
